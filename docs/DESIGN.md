# 乐学小岛（lexuedao）— 全局设计方案

> 语数英全科学习微信小程序 · 游戏化 · 竞技 · AI 推荐 · 订阅制 · 跨端

---

## 一、产品定位

让小学生像玩游戏一样主动学语文、数学、英语。

**核心理念：「学-练-战-奖」四环驱动**

```
学（获取知识）→ 练（巩固技能）→ 战（竞技验证）→ 奖（激励反馈）→ 循环
```

孩子每完成一个环，AI 分析薄弱点，推荐下一环内容，形成个性化学习闭环。

- 学习 = 闯关，练习 = 打怪，考试 = Boss战
- 每科有独立的「冒险地图」，三科共享「角色成长体系」
- 同学组队、PK、排行榜，社交驱动学习动力
- AI 智能推荐薄弱知识点，精准练习
- 家长实时查看学习报告，了解孩子薄弱环节

## 二、技术架构

| 层级 | 技术 | 理由 |
|------|------|------|
| 小程序框架 | Taro 4.x + React 18 | 现有技术栈，支持编译到 Android/iOS App |
| 状态管理 | Zustand（分 Store） | 轻量，分包按需加载 |
| UI 样式 | SCSS + CSS Modules | 隔离性好 |
| 后端 | CloudBase 云函数 (Node.js 18) | 零运维，原生微信鉴权 |
| 数据库 | CloudBase PostgreSQL 15+ | 关系型 + 窗口函数 + pgvector |
| AI 推荐 | pgvector + 云函数 + 大模型 | 语义向量检索 + LLM 对话 |
| 存储 | CloudBase 云存储 | 图片、音频 |
| 支付 | 微信虚拟支付 + 微信支付 | 小程序原生 |
| 跨端 App | Taro RN + react-native-skia | Canvas 书写适配 |

### 为什么选 PostgreSQL 而不是 MySQL？

- **窗口函数**：排行榜（总分榜/单科榜/本周/本月/班级/全国）一条 SQL 搞定，MySQL 需要多次子查询
- **pgvector**：AI 学习推荐的向量检索基础
- **CloudBase 官方趋势**：2026 年重点推 PG

## 四、学生端 vs 家长端

**一个小程序，角色切换模式**（不是两个独立小程序）

| 维度 | 学生模式 | 家长模式 |
|------|------|------|
| 首页 | 冒险地图 + 今日任务 + AI推荐 | 学习报告 + 多娃切换 + 薄弱分析 |
| 学习 | 闯关练习 + 对战 + 排行 | 查看进度 + 错题分析 + 推荐练习 |
| 我的 | 角色装扮 + 金币商城 | 娃管理 + 订阅管理 |

切换机制：底部「我的」→「切换身份」，家长切学生需输入密码。

### 6.1 角色成长
- 等级 Lv.1~50（全科共享经验值）
- 金币 💰（练习获得，兑换皮肤）
- 钻石 💎（稀有货币，成就/付费获得）
- 体力 ⚡（每日上限，防沉迷）

### 6.2 段位系统
- 青铜 → 白银 → 黄金 → 铂金 → 钻石 → 王者
- 每学科独立段位，每赛季重置
- 对战胜利获得段位分

### 6.3 成就徽章（23个）
- 通用：初次见面、七日坚持、等级成就、收藏家
- 语文：初学乍练、百字达人、书法大师、千锤百炼
- 数学：算术新星、速算高手、满分学霸、万题之王
- 英语：单词新手、词汇达人、拼写冠军、听力高手
- 对战：首战告捷、十连胜、百战勇士
- 全科：全科小状元

### 6.4 每日挑战
每天刷新3个任务（语/数/英各一），完成领奖励

### 6.5 排行榜
- 总分榜 / 单科榜
- 班级 / 全校 / 全国（同年级）/ 好友
- 利用 PG 窗口函数（ROW_NUMBER）高效实现

### 6.6 对战系统
- 好友对战：邀请微信好友同屏PK
- 随机匹配：同年级在线玩家
- 3轮题目，积分制

### 6.7 冒险地图（学科首页）
每个学科首页是可视化闯关地图，替代传统列表

### 6.8 宠物/伙伴系统（后期）
- 学习获得经验喂养虚拟宠物
- 宠物在对战中提供加成
- 收集、进化、装扮

│       │            │               │             │
│       ▼            ▼               ▼             │

│  │  ┌─────────┐ ┌──────────┐ ┌─────────────┐  │ │
│  │  │ 薄弱标签 │ │ 能力向量  │ │ 错题特征聚类 │  │ │

│                       │                           │
│                       ▼                           │

│  └─────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────┘

### 7.1 分阶段实现

| 阶段 | 功能 | 技术方案 |
|------|------|----------|
| V1 | 基于错题本的规则推荐 | PG 查询错题标签 → 匹配同标签题目 |
| V2 | 向量相似度推荐 | pgvector 余弦相似度检索薄弱知识点 |
| V3 | AI 学习助手聊天 | CloudBase 大模型（DeepSeek/Hunyuan）流式对话 |
| V4 | 拍照搜题/改错 | OCR + 大模型识别 + 题库匹配 |

### 7.2 推荐场景

| 场景 | 触发条件 | 推荐内容 |
|------|------|------|
| 每日推荐 | 每天首次打开 | 基于薄弱点的3-5个练习 |
| 错题重练 | 错题本有未掌握项 | 同类知识点的变式练习 |
| 闯关推荐 | 某关卡反复失败 | 前置知识点的巩固练习 |
| 家长报告 | 家长模式查看 | 薄弱点分析 + 推荐练习方案 |

### 7.3 数据表

- `knowledge_vectors`：知识点向量表（含 256 维语义向量、难度、标签）
- `user_skill_profile`：用户能力画像表（能力向量、薄弱/擅长标签）
