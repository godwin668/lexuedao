# 乐学小课堂（lexue-weapp）— 全局设计方案

> 小学生语数英全科学习微信小程序 · 游戏化 · 竞技 · 订阅制

---

## 一、产品定位

让小学生像玩游戏一样主动学语文、数学、英语。

- 学习 = 闯关，练习 = 打怪，考试 = Boss战
- 每科有独立的「冒险地图」，三科共享「角色成长体系」
- 同学组队、PK、排行榜，社交驱动学习动力

---

## 二、技术架构

| 层级 | 技术 | 理由 |
|------|------|------|
| 小程序框架 | Taro 4.x + React 18 | 现有技术栈，支持多端 |
| 状态管理 | Zustand（分 Store） | 轻量，分包按需加载 |
| UI 样式 | SCSS + CSS Modules | 隔离性好 |
| 后端 | CloudBase 云函数 (Node.js) | 零运维，原生微信鉴权 |
| 数据库 | CloudBase MySQL | 关系型，JOIN/事务/外键 |
| 缓存 | CloudBase 文档型 DB | 排行榜、对战临时数据 |
| 存储 | CloudBase 云存储 | 图片、音频 |
| 支付 | 微信虚拟支付 + 微信支付 | 小程序原生 |

### 为什么不用自建 PostgreSQL？

- 项目规模不需要 PG 高级特性
- CloudBase 零运维，专注业务
- 原生微信鉴权，免 OAuth
- 初期基本免费

## 四、学生端 vs 家长端

**一个小程序，角色切换模式**（不是两个独立小程序）

| 维度 | 学生模式 | 家长模式 |
|------|------|------|
| 首页 | 冒险地图 + 今日任务 | 学习报告 + 多娃切换 |
| 学习 | 闯关练习 + 对战 + 排行 | 查看进度 + 错题分析 |
| 我的 | 角色装扮 + 金币商城 | 娃管理 + 订阅管理 |

切换机制：底部「我的」→「切换身份」，家长切学生需输入密码。

## 五、游戏化体系

### 5.1 角色成长
- 等级 Lv.1~50（全科共享经验值）
- 金币 💰（练习获得，兑换皮肤）
- 钻石 💎（稀有货币，成就/付费获得）
- 体力 ⚡（每日上限，防沉迷）

### 5.2 成就徽章（23个）
- 通用：初次见面、七日坚持、等级成就、收藏家
- 语文：初学乍练、百字达人、书法大师、千锤百炼
- 数学：算术新星、速算高手、满分学霸、万题之王
- 英语：单词新手、词汇达人、拼写冠军、听力高手
- 对战：首战告捷、十连胜、百战勇士
- 全科：全科小状元

### 5.3 每日挑战
每天刷新3个任务（语/数/英各一），完成领奖励

### 5.4 排行榜
- 总分榜 / 单科榜
- 班级 / 全校 / 全国（同年级）/ 好友

### 5.5 对战系统
- 好友对战：邀请微信好友同屏PK
- 随机匹配：同年级在线玩家
- 3轮题目，积分制

### 5.6 冒险地图（学科首页）
每个学科首页是可视化闯关地图，替代传统列表
